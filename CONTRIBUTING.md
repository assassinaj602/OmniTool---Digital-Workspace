## ğŸ› ï¸ How Can I Contribute?

### 1. Reporting Bugs
*   Check if the bug has already been reported in the [Issues](https://github.com/yourusername/omnitool/issues).
*   Use the **Bug Report** template.
*   Include screenshots and reproduction steps.

### 2. Suggesting Features
*   We love new tool ideas!
*   Use the **Feature Request** template.
*   Explain *why* this tool fits the "Client-Side Privacy" philosophy.

### 3. Code Contributions
1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  Create a **Branch** for your changes.
    *   `feat/new-tool-name`
    *   `fix/resize-bug`
    *   `docs/update-readme`
4.  **Commit** your changes using conventional commit messages.
    *   `feat: add pdf to gif converter`
    *   `fix: resolve mobile layout overflow`
5.  **Push** to your fork.
6.  Open a **Pull Request**.

## ğŸ¨ Code Style & Standards

*   **Framework:** We use React 19 functional components with Hooks.
*   **Language:** Strict TypeScript. No `any` unless absolutely necessary.
*   **Styling:** Tailwind CSS. 
    *   Use utility classes over custom CSS.
    *   Use `dark:` modifiers for dark mode compatibility.
*   **Typography:** 
    *   Use `font-heading` for titles (Satoshi).
    *   Use `font-sans` for UI (Inter).
    *   Use `font-mono` for technical data (JetBrains Mono).
*   **Icons:** Use the `Icons.tsx` component library.
